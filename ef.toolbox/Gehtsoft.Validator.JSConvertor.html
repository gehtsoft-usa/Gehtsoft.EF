Namespace Gehtsoft.Validator.JSConvertor

Brief

Conversion of the validator to JavaScript

Details

The namespace provides basic tools to create JavaScript validator on the base of C# validator.

The functionality can be used, for example, to provide client-side validation on the WEB site.

Please note that in order to be able to use this feature you must:

1) Add Gehtsoft.ExpressionToJs package to your project

2) Add the scripts returned by ExpressionToJsStubAccessor.GetJsIncludesAsString() method into the scripts on the page where validation is supposed to be used.

No all predicates may be correctly ported. Only following predicates are successfully compiled to the JavaScript:

  • Null

  • NotNull

  • NotNullOrEmpty

  • ShorterThan and all predicates based on ShorterThan (e.g. MustHaveValidDbSize for strings)

  • Between and all predicates based on Between (e.g. MustBeInDbValueRange)

  • Match and all predicates based on Match (e.g. Email)

  • The LINQ predicates that uses only:

    • Access to the entity or a value

    • +, -, *, /, % math operators

    • ==, !=, >, <, >=, <= comparison operators

    • &, ||, ! boolean operators

    • The following string methods and properties: Length, Upper, Lower, Trim, StartsWith, IndexOf, Substring, IsNotNullOrEmpty, IsNotNullOrWhitespace

    • The IsMatch method of RegExp

    • The properties and methods of Math class.

    • Year, Month, Day, DayOfWeek, Hour, Minute and Second properties and AddDays, AddHours, AddMinutes, AddSeconds method of DateTime

    • TotalMilliseconds, TotalSeconds, TotalMinutes, TotalHours and TotalDays properties of TimeSpan

    • + and - operator between DateTime and TimeSpan.

    • - operator between two DateTime

    • Functions in Gehtsoft.ExpressionToJs.Functions class

    • HasValue and Value properties of Nullable

    • IsUpper, IsLower, IsDigit, IsLetter, IsLetterOrDigit, IsPunctuation and IsWhitespace methods of Char.

    • Any, All, Empty, Count, First and Last with lambda predicate for lists, arrays and strings.

In other words, function, enum value validity, uniqueness check, existence check as well as any expression predicate which is not supported by the compiler won't be compiled into JavaScript code.

Please note that expression predicate compilation will case the exception in case of anything what is not supported is found, unless the whole rule is marked as a server-only rule.

You can also provide your own implementation of the expression compiler derived from the standard one in order to support more features for expression predicates.
