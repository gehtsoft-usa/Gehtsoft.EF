Parents

object

Brief

The controller to create, drop or update entities automatically.

Details

The method recognizes and handles the following cases:

  • Create table for the entity if the table does not exist.

  • Drop table for the entity if the table is attributed using ObsoleteEntityAttribute

  • Add column for the entity if the column does not exist.

  • Drops column if the column attributed using ObsoleteEntityPropertyAttribute

If you need to enrich the operation with the custom logic, you can use action (see EntityActionDelegate ) or events CreateEntityController.OnAction .

Note: Some connections do not support dropping of the columns. Check SqlDbLanguageSpecifics.DropColumnSupported flag to verify the connection. If connection does not support columns dropping, the columns will remain in the table.

Note: The controller does not recognize the situations when:

  • The type, name or other parameters of the column has changed.

  • The property deleted from the entity.

  • A new index is added via ICompositeIndexMetadata

  • View changed.
