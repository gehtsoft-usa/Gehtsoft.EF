Declaring Entities

What is Entity?

In the entity framework an entity is supposed to be ... an entity, in other words a special kind of a class which keeps only data and does not have any methods.

How to Define Entity?

The entity which is good for an entity framework must:

  • be declared as a public class

  • have parameterless constructor

  • have entity data declared as public properties with get and set accessors defined.

The entity is attributed using EntityAttribute. The attribute sets the table name.

Example Show

Example Hide

[Entity(Table = "entity_table")]
public class Entity
{
}

How to Define Property?

The properties are attributed using EntityPropertyAttribute. In the simplest case nothing additional needs to be set.

Example Show

Example Hide

[Entity(Table = "entity_table")]
public class Entity
{
    [EntityProperty()]
    public int IntegerProperty { get; set; }
}

In this case the property name, type and parameters are all taken from the property definition.

When To Configure Property Size?

For some types the property size cannot be get from the property definition (e.g. for strings) or may be not effective if it is taken from the property definition (e.g. numbers).

In this case Size and Precision (for numbers) attribute properties can be utilized.

Example Show

Example Hide

[Entity(Table = "entity_table")]
public class Entity
{
    [EntityProperty(Size = 64)]
    public string StringProperty { get; set; }
 
    //8-digit number, 2 digits after decimal point
    [EntityProperty(Size = 8, Precision = 2)]
    public string NumericProperty { get; set; }
}

When To Set Property Type?

For some types the appropriate database type cannot be get from the property definition. For example, C# for not have special types for date and timestamps, but the most of database do.

In this case the DB type needs to be specified.

Example Show

Example Hide

[Entity(Table = "entity_table")]
public class Entity
{
       [EntityProperty(DbType=DbType.Date)]
       public DateTime DateProperty { get; set; }
}

How to Make a Property Nullable?

To make a property nullable, set Nullable property of the attribute to true. Also, if the property type is a value type, change the property type to nullable.

Example Show

Example Hide

[Entity(Table = "entity_table")]
public class Entity
{
       [EntityProperty(DbType=DbType.Date, Nullable=true)]
       public DateTime? DateProperty { get; set; }
}

How to Add an Index to the Property?

To add an index to the property use Sorted property of the attribute.

Example Show

Example Hide

[Entity(Table = "entity_table")]
public class Entity
{
       [EntityProperty(DbType=DbType.Date, Sorted=true)]
       public DateTime? DateProperty { get; set; }
}

How to Customize Database Field Name?

If you want to customize the database field name use Field property of the attribute.

Example Show

Example Hide

[Entity(Table = "entity_table")]
public class Entity
{
       [EntityProperty(Field="action_date", DbType=DbType.Date, Sorted=true)]
       public DateTime? DateProperty { get; set; }
}
