Primary Keys

The entity framework does not support complex primary keys. Only one property can be a primary key of the entity.

To mark the property as a primary key use PrimaryKey property of the attribute. The property of any type except properties mapped to BLOB and CLOB columns can be declared as primary key.

When a property is declared as a primary key, it will be used for:

  • Updating the entity in the database

  • Loading a single instance of the entity from the database

  • Establishing relationship with other entities

For integer artificial primary keys the Autoincrement property can be used to create an sequence that starts from 1 and increases by one every time when a new entity is saved to the database. This operation is atomic at the database level and uniqueness of the values of guaranteed.

Example Hide

[Entity(Table = "entity_table")]
public class Entity
{
    [EntityProperty(PrimaryKey = true, Autoincrement = true)]
    public int PK { get; set; }
}

A shortcut for and integer primary key with autoincrement is AutoId

Example Hide

[Entity(Table = "entity_table")]
public class Entity
{
    [EntityProperty(AutoId = true)]
    public int PK { get; set; }
}

Please note that autoincrement sequences ALWAYS start from 1 for each entity individually, so you can use PK == 0 condition to check whether you are dealing with a new entity.
