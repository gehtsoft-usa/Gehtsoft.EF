Foreign Keys

Refer to Other Table

To create a reference to other entity:

  • Create a property that has the type of the other entity.

  • Mark it with EntityProperty attribute with ForeignKey property set to true.

In the case, the column with the type of the primary key of the other entity will be added to this table and the foreign key constraint will be created.

When 0..* relationship needs to be established, use Nullable property of the attribute.

Example Hide

[Entity(Table = "dictionary_table")]
public class Dictionary
{
    [EntityProperty(AutoId = true)]
    public int ID { get; set; }
 
}
 
 
[Entity(Table = "entity_table")]
public class Entity
{
    [EntityProperty(AutoId = true)]
    public int ID { get; set; }
 
    [EntityProperty(ForeignKey = true, Nullable = true)]
    public Dictionary Dictionary { get; set; }
}

Self-References Table

You can make self-reference to the entity in order to create a tree table.

Please note that:

  • The list reference must ALWAYS be nullable.

  • MySQL does NOT support tree queries, so using tree tables will exclude MySQL and MariaDb from the list of the DB that can be used in the application.
